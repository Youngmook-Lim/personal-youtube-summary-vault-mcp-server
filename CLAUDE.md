# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Teaching Mode — Read This First

The user is learning MCP by building this project. **Do not write code automatically.** Default behavior is to give step-by-step instructions so the user can do it themselves. Only write code when the user explicitly asks. When in doubt, explain what to do and why — let the user execute it.

## Project Overview

A local Python MCP server that turns a private Obsidian GitHub repo (filled with YouTube video summaries generated by an n8n pipeline) into queryable, synthesizable context available to both Claude Desktop and Claude Code.

Equal goals: a working server used daily, AND deep understanding of why MCP exists and what it enables beyond a custom RAG app.

**User context:** Junior Java/Spring developer in Seoul, MacBook Pro M1. Has prior ChromaDB experience. Corpus is Korean + English.

## Canonical Source Rule

Before writing any MCP code, decorators, resource URIs, prompt schemas, or client config — consult:

- `https://github.com/modelcontextprotocol/python-sdk` (official Python SDK README + examples)

**These sources override everything else in this file.** When anything here conflicts with current docs, the docs win. State the discrepancy explicitly when found.

## Architecture

| Concern | Decision |
|---|---|
| Language / SDK | Python 3.11+, official `mcp` SDK (FastMCP) |
| Transport | stdio, local only (no SSE, no auth in v1) |
| Metadata + keyword search | SQLite with FTS5 |
| Vector store | `sqlite-vec` preferred; ChromaDB as fallback |
| Embeddings | Multilingual (KO + EN). Candidates: OpenAI `text-embedding-3-small`, Gemini `text-embedding-004`, `BAAI/bge-m3`. Decide in step 1. |
| Source of truth | Private Obsidian GitHub repo, cloned locally |
| Sync | Manual `reindex_library()` tool only — no file watching |
| Clients | Claude Desktop + Claude Code |

## Database Schema

```sql
videos(id, file_path, file_hash, title, channel, url, published_at, ingested_at, frontmatter_json, summary_md)
chunks(id, video_id, chunk_index, content, embedding_ref)
tags(video_id, tag)
-- FTS5 virtual table over videos.summary_md and chunks.content
```

## MCP Primitives to Register

**Tools**
- `reindex_library()` — git pull + ingest new/changed files (idempotent by content hash)
- `search_videos(query, semantic=True, limit, channel?, date_range?)`
- `find_related(video_id, n=5)`
- `synthesize_across(theme, max_videos=10)`
- `get_video(video_id)`
- `list_recent(days=14)`
- `suggest_forgotten(min_age_days=30)`

**Resources**
- `video://{id}`, `library://recent`, `library://forgotten`, `channel://{name}`, `tag://{tag}`

**Prompts**
- `/synthesize-theme {theme}`, `/whats-related {topic}`, `/revisit-suggestion`, `/cite-from-library {topic}`

## Build Steps

1. **Bootstrap** — Verify Python version, install `mcp[cli]` + chosen vector/embedding deps, scaffold project.
2. **Schema** — Create SQLite tables + FTS5 virtual table.
3. **Indexer module** (pure Python, no MCP yet) — git pull → scan subfolder → parse frontmatter + body → hash → skip unchanged → chunk (~500 tokens, small overlap) → embed → write to SQLite + vector store. Log added/skipped/updated.
4. **MCP server** (FastMCP) — Register tools, resources, prompts from the list above.
5. **Wire clients** — Add server entry to Claude Desktop and Claude Code MCP configs (verify current schema + file paths against canonical docs). Document restart cycle.
6. **Pedagogical validation** — Run one flow per MCP primitive to demonstrate why each exists distinctly (tool approval UI, resource attachment vs. tool call, prompt slash menu, reflection writeup).
7. **Document gotchas** — stdio buffering, JSON-RPC errors, resource URI rules, tool input schema validation, what context a client attaches when a resource URI is referenced.
